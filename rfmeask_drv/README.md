# Прошивка контроллера синхронизации

Программа управления платой-синхронизатором контроллера сканирования.

Плата занимается контролем положения изделия путем генерации управляющей и приёма контольной последовательности импульсов управления СУ.
Программа способна обрабатывать прерывания при достижении изделием определенных положений, обрабатывает и отсылает импульсы синхронизации внешней измерительной аппаратуре.

Команды управления принимаются по каналу RS232. см. rs232.cpp

## Режимы работы

waittrigmode = 1. 
	Каждый раз после отправки триггера синхронизации.
	Изделие будет ожидать ответный триггер от устройства.

waittrigmode = 0. 
	В этом режиме изделие не посылает и не считывает
	триггеры, а сразу отсылает уведомление.

manual_trig_mode 
	Устанавливает отсылку сообщения о принятой точке непосредственно на
	приём триггера. (без логики контроллера движения)

## Выводы

Вывод TRIGOUT инвертирован. 
В неактивном состоянии его уровень - логическая 1 - 3.2В. (Существует команда смены полярности)
